# CDAC_17b (oref DAC) 검증 스펙 & 결과

**날짜**: 2026-07-02 | **공정/코너**: Samsung 28FDS, TT / 27°C / presim(cmom device model 포함) | **DUT**: `0_BARAM/CDAC_17b` (풀칩 RC_network 내, VOUT=oref)

## 1. 대상
3-세그먼트 split-CDAC, oref 생성(DLF acc → oref). 단위캡 cmom 4.436fF, weight=mult.
- **VX seg** bits0-4(1,2,4,8,16) + 터미네이터 C13 + 브리지 C12(VM-VX)
- **VM seg** bits5-9(1,2,4,8,16) + 브리지 C14(VOUT-VM) [+ VM 터미네이터 C_VM — 원설계 누락, 추가함]
- **VOUT seg** bits10-16(1,2,4,8,16,32,64) = 출력=oref

## 2. 검증 방법
`cdac17_test/build_cdac17_tb.il` (buildCdac17TB, 12b 방식). 비트0-10 카운터=코드0-2047(경계 32,1024 포함), 비트11-16=0, VOUT/VM/VX 리셋SW3. tran 41µs(Tcode20n), export time,VOUT → 코드창 슬라이스 INL/DNL.
+ 이상 전하모델(Python, 12b로 검증됨: redundant 155→err0)로 사전 예측.

## 3. 진단·수정 경과
1. **이상분석**: 원설계(C12=4,C13=96,C14=4) 비이진 예측 → **Spectre 확정**(code1023→1024 −20.6mV 급락, INL±624/DNL−1209).
2. **원인**: (a) VM 터미네이터 부재, (b) 브리지 값/배치 오류.
3. **구조 수정**: VM 터미네이터 추가 + 브리지 재배치(VOUT-VM 작게, VX-VM 크게) → 단조 복원.
4. **cmom 기생 반복튜닝**(12b 155→143과 동일): 이상값→실측 편차(VM/VOUT 과대)를 2점 보정으로 기생모델화(p_node=a·node_mult, aVX=aVM≈0.065, aVO≈0.102) → 정수값 수렴.
   - 수렴: INL ±395 → ±7.3 → **±1.42** LSB.

## 4. 결과 (정수 최종)
**C12(VX-VM)=11, C14(VOUT-VM)=6, C13(VX-VSS)=279, C_VM(VM-VSS)=133** (116279133.csv)
| 항목 | 값 |
|------|-----|
| INL | −1.40 / +1.42 LSB |
| DNL | −0.97 / +1.05 LSB |
| 단조성 | 완전 단조 |

- **정수 양자화 바닥 ~1 LSB**: 경계서 VOUT 1유닛≈6 LSB라 정수로 그 이하 불가. C_VO(VOUT 터미네이터) 추가해도 ~1 LSB(개선 미미) → 생략.
- <0.5 LSB는 **fractional/trim 캡**(cmom 비정수 실현시 연속최적 C12≈10.9/C14=6/C13≈285/C_VM≈133/C_VO≈51 → ~0) 필요. 우선 정수배 채택.

## 5. 결론 & 남은 것
- **정수 설계 종결**: INL/DNL ±1.4 LSB, 단조. oref는 **DLF 적분루프 기준**이라 INL은 루프가 흡수, **단조성이 핵심** → 충분.
- ⚠ **잔여: MC(mismatch)로 −DNL 마진 확인** — DNL min −0.97가 −1 근접, mismatch서 <−1(missing-code) 안 되는지. carry코드(특히 code1024) DNL 3σ 확인.
